# Workflows

这组投研 skills 可以组合成类似 gstack 的研究流程。

## 最低操作量入口

- `sk-autopilot`
  默认总控入口。适合不想自己挑 skill 的使用者。

## 推荐顺序

1. `sk-autopilot`
   用于自动判断应该走哪条研究链路。
2. `sk-thesis`
   用于定义问题和值不值得研究。
3. `sk-industry-map`
   用于搭行业框架和产业链地图。
4. `sk-company-deepdive`
   用于下沉到公司层面。
5. `sk-earnings-preview`
   用于财报前瞻和财报后验证。
6. `sk-model-check`
   用于检查盈利预测和估值模型。
7. `sk-consensus-watch`
   用于判断市场预期差。
8. `sk-catalyst-monitor`
   用于跟踪财报、政策和产业催化。
9. `sk-roadshow-questions`
   用于准备路演和管理层交流问题。
10. `sk-red-team`
   用于强制反方审视。
11. `sk-pm-brief` 或 `sk-briefing`
   用于输出给基金经理、晨会或团队流转。

## 团队映射

- `sk-autopilot`：总控调度入口
- `sk-thesis`：基金经理 / 投资主理人
- `sk-industry-map`：行业研究员
- `sk-company-deepdive`：公司研究员
- `sk-earnings-preview`：财报季前瞻助理
- `sk-model-check`：模型审阅助理
- `sk-consensus-watch`：预期管理助理
- `sk-catalyst-monitor`：事件跟踪助理
- `sk-roadshow-questions`：调研提纲助理
- `sk-red-team`：反方风险官
- `sk-pm-brief`：面向投资决策层的输出官
- `sk-briefing`：晨会与材料整理助理
